not_rocketscience.background module

Everything related to the scrolling background for Not Rocketscience. Scrolling background effect is achieved by initializing multiple layers of starry backgrounds (ScrollingStarBackground), which are then scrolled with different speeds. This generates a fake 3D effect.

class not_rocketscience.background.LayeredScrollingStarBackground(tile_size: tuple, initial_position: array, space_color=(10, 10, 30, 255), star_color=(255, 255, 200, 255), n_stars=10, n_layers=3)

Bases: object

Container class to layer multiple instances of ScrollingStarBackground on top of each other. Layers will be enlarged by factors defined in LayeredScrollingStarBackground.scale_modifiers and movement speeds will be changed by LayeredScrollingStarBackground.speed_modifiers to create a parallactic effect.

Parameters:

  • tile_size – unmodified tile size of the back-most layer. This layer will also have opaque background.

  • initial_position – Initial screen position of the initial tiles on all layers.

  • space_color – RGB tuple for color drawn as background of the back-most layer

  • star_color – RGB tuple for color of stars drawn on all layers

  • n_stars – amount of stars on each layer

  • n_layers – amount of layers

draw_tiles(screen, ship_position, dt, speed)

Draws the tiles for all layers, see ScrollingStarBackground.draw_tiles(). The factors in LayeredScrollingStarBackground.speed_modifiers are applied to the speed vector on each layer.

Parameters:

  • screen – Display surface of pygame.

  • ship_position – 1 x 2 numpy.array position towards which tile positions will be evaluated.

  • dt – time step size, usually the time passed during one frame.

  • speed – 1 x 2 numpy.array representing speed vector. tiles wil move in opposite direction to simulate movement.

property scale_modifiers

Foreground layers appear bigger by the factor indicated in this list.

property speed_modifiers

layer movement speeds are reduced by a quadratically reducing factor for each layer index

class not_rocketscience.background.ScrollingStarBackground(tile_size: tuple, initial_position: array, space_color=(10, 10, 30), star_color=(255, 255, 200), fill_background=True, star_size=2, n_stars=10)

Bases: object

One layer of scrolling star backgrounds. The background color can be opaque or transparent to layer multiple star backgrounds on top of each other. One layer of background is tiled, i.e. if the center of attention (usually ship position) shifts into a certain direction, new tiles with random stars are generated in that direction. Tiles that have shifted away far enough are removed. Far enough means 3 times tile size in city block metric distance to center of attention.

Parameters:

  • tile_size – Size in pixels of one tile of starry background

  • initial_position – center position in screen coordinates of first tile

  • space_color – background color. Only shows if fill_background is True

  • star_color – color which stars are drawn in

  • fill_background – fill background with space_color (True default), if False transparent

  • star_size – Size in pixels of ‘square’ stars

  • n_stars – amount of stars in one tile.

draw_tiles(screen, ship_position, dt, speed)

After deciding which tiles need to be removed and which need to be added based on ship_position and speed, shift all tiles by the distance travelled during time step dt with speed into the opposite direction, and then draw them on the screen.

Parameters:

  • screen – Display surface of pygame.

  • ship_position – 1 x 2 numpy.array position towards which tile positions will be evaluated.

  • dt – time step size, usually the time passed during one frame.

  • speed – 1 x 2 numpy.array representing speed vector. tiles wil move in opposite direction to simulate movement.

manage_tile_positions(pos, speed)

Dynamically adds tiles as neighbors to tile closest to screen coordinate pos, in the direction pointed by speed.

Parameters:

  • pos – 1 x 2 numpy.array representing a screen coordinate, to determine closest tile

  • speed – 1 x 2 numpy.array indicating the speed vector of movement to determine direction in which tiles need to be added

property tiles

Manages the tiles by id, based on coordinates in ScrollingStarBackground.positions. For tile ids in positions, which are not yet in the _tiles dictionary, a new tile is added. Tile ids which are in _tiles but not in positions any more are deleted.
